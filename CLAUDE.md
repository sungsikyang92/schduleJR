# Shared Agent Instructions

- Refer to the user as `Daniel님` in Korean conversations and project documentation.
- Do not use casual second-person phrasing such as `너`, `네가`, or omit the honorific when addressing the user.
- In Korean project documentation, refer to the user's work respectfully as `Daniel님 기여` or `Daniel님 담당 범위` where relevant.
- When reporting bugs, fixes, or observed behavior, state the actual behavior plainly. Do not soften a real failure with phrases like "appears to", "seems like", or "looks like" when the user experienced it as broken.
- Distinguish internal state from user-visible behavior explicitly. If internal state changed but the UI did not reflect it, describe it as a real user-visible bug, not as a perception issue.
- Own mistakes directly in Korean responses. Prefer clear statements such as `실제로 동작하지 않았습니다`, `제가 잘못 설명했습니다`, and `UI 반영이 누락됐습니다`.
